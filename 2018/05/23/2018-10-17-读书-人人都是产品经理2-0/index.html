基本信息

作者：苏杰

阅读日期：2018/3/28 - 2018/4/8 初读

阅读摘要

互联网行业发展迅速，产品经理也随之变成一个“热门话题”。这本书从产品经理的思维特点入手，结合一个产品从无到有的整个过程，讲述了如何从一个“泛产品经理”成长为专业的产品经理，并且介绍了与产品经理关联度很大的工作——运营，最后谈到产品经理的个人职业发展前景。正文后面有作者推荐的图书、公众号等资源。

核心问题

Q1 主要内容是什么

学生思维与PM思维。有一个例子——一辆运送西瓜的火车遇到一个限高的山洞，过不去了，怎么办？学生思维可能就一上来开始想怎么解决这个问题，而PM的素养是，思考这个问题背后的问题，比如这次的运送是偶尔的还是经常发生的，不同解决方法的效益如何，可不可以直接舍弃掉一部分西瓜。学生思维下的问题大多是都是固化的，至少是理想化的，而实际的问题则需要不仅仅盯住问题本身，还要关注到与“问题”有关的一系列问题，这样才能得出一个在现实中可行且合理的解决方案。

关于产品的定义——“解决某个问题的东西”，在分析一款产品的时候，要看它解决了什么问题，这个问题的最具体形式是什么，解决方案是什么。由此可联想到，还有没有更好的解决办法。

关于Y模型。Y的左分支 = 用户的要求，右分支 = 产品功能，一竖 = 人性和价值观，交叉点 = 需求背后的动机或目标。在听到用户的反馈与要求时，至少要做到分析出这个要求背后的动机，然后根据这个动机来确定产品的功能，而不是用户说想要什么功能就做什么功能。最理想的情况是，在发掘出交叉点处的动机之后，还能继续深入下去，挖掘出动机背后反映出的人性层面的需求。这里链接到马斯洛需求层次理论。

关于“攀谈术”。在向用户了解需求的时候，经常会出现用户无法正确表达自己需求的情况，这时候就需要访问者改变访问方式。例如假设没有这个功能会怎么样，帮助他回忆最近一次与产品相关的场景（有了具体的场景之后，人们会更容易描述自己的感受与情感）。这个技巧不仅仅可以用在访问用户需求上，在平时与人聊天的过程中，特指交流比较深入的时候，似乎也可以使用这种技巧，让对话的深度上升一级。

关于“放弃”。在打造一款产品的时候，更多的时候是放弃掉很多功能。（这一点与我平时的思维还是很不一样的）

PM的七个层次。从泛产品经理到产品经理，到产品总监，到CEO，到最后上升到人文情怀、造福世界的境界。漫漫“修仙”路。一般悟性较好的新人可以在工作两年左右之后，可以开始负责一个完整的产品。

Q2 有什么感想

预期之外的收获。本意是了解PM这份工作，最后更大的收获来自于：对PM思维方式的理解，对产品的理解，以及关于未来的可能性又多了一个方向。

此书的逻辑结构有待完善。在整理思维导图的时候发现，这本书的整理脉络看起来很清晰，但是每一个章节里面的逻辑还有不完善之处，会出现那种跟本章内容无甚关联的part出现。

Q3 有什么收获

培养时间大局观的技巧。研究明星产品的更新日志，有机会可以与找对应的PM交流验证。

逻辑提取能力得到锻炼。在阅读文档的压力下，每读完几个章节之后，我都需要去整理其中的脉络结构，一开始还比较艰难，后来就开始有得心应手的感觉了，感觉像是熟悉了一项技能。这个技能也帮助我，在平时碎片化阅读的时候，能够更好的吸收所读内容。哈，开心。

章节图示有可取之处。这本书的每一章后面都自带总结用的思维导图，有一些符号很生动的表示事物间的联系，我在绘制理解导图的时候，有刻意模仿。不过这些图稍显花哨，有点让人一下子抓不住重点。

Q4 什么时候读

前五章 → 未入门或即将入门的PM，用于了解PM的思维方式，搭建一个体系

6~8章 → 已入门的PM，具体做到某项工作时参阅，做完这项工作时再结合对应文本内容进行复盘总结。这几章介绍很多了详细的工作流程、工具、方法等，可用作每项细分工作的说明书与参考书。

9~11章 → 本职工作已经驾轻就熟的PM，在思考发展方向时参阅

12章 → 推荐书籍、公众号。另外，每一章节后面也都附有推荐书目，也可以整理成书单。

阅读方法论

  • 微信读书的 划线 功能非常好用，有三种划线形式（高亮、直线、波浪线），对应不同性质的文本，划线内容收集在侧边栏中，打开侧边栏，点击划线内容，可以跳转到原文。完美的解决了在阅读中记笔记的问题

  • 找到与阅读“合拍”的场景。场景可以是一段时间、一个地点、一个情境，最好是一个画面，例如玉泉图书馆一楼的大沙发，面前是通透的大型玻璃窗，窗外是怡人的春景；室友都还在睡觉的清晨，窗外只能听见鸟叫，台灯晕出柔软的暖黄色光。在这样的场景中阅读，专注力和幸福感都 up up up 呀！

  • 写阅读文档时，注意总结小标题

  • BGM推荐 《The Good Wife (Theme)》激昂澎湃，根本停不下来
